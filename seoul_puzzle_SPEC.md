## 1. 프로젝트 개요

한국인 친구 **김연세**와 서울을 함께 여행하는 설정의 모바일 웹 한국어 학습 게임. 어절 카드를 순서대로 탭해 문장을 조립하는 퍼즐 방식으로 문법과 어휘를 자연스러운 맥락 속에서 익힌다.

| 항목 | 내용 |
|---|---|
| 장르 | 어절 조립 퍼즐 (문장 완성형) |
| 플랫폼 | 모바일 웹 (최대 너비 390px) |
| 교재 기준 | 연세한국어 1권 4~15과 |
| 총 장소 수 | 10개 |
| 총 문제 수 | 37문제 (장소별 3~4문제) |
| 구현 방식 | React 18 + TypeScript (단일 TSX 파일) |
| 데이터 | `data/seoul_puzzles.json` (분리 관리) |
| 진행 저장 | localStorage (`seoul-puzzle-v1`) |

## 2. 학습 목표 및 대상

### 대상 학습자

- 한국어를 막 시작한 초급 학습자 (연세한국어 1권 4~15과 수준)
- 교실 밖 실제 맥락에서 한국어를 쓰고 싶은 학습자
- 단순 암기보다 스토리·맥락 기반 학습을 선호하는 20대

### 학습 목표

- 각 과의 핵심 문법 항목을 **상황 속에서** 반복 노출
- 어절 단위 조립을 통해 **어순 감각** 내재화
- 한국인 친구와의 대화라는 설정으로 **구어체 표현** 습득

### 어휘·문법 수준 원칙

- 해당 장소의 퍼즐은 **해당 과 이하** 문법·어휘만 사용
- 친구(김연세)의 대사도 동일 수준 이하로 제한
- 오답 선택지는 정답 카드와 조합해도 올바른 문장이 되지 않는 것만 허용
- **과거형(`었/았`) 금지** — 1급 범위는 현재형 중심 (과거형은 연세 2급 9과 도입)

## 3. 게임 설정 및 세계관

### 등장인물

| 인물 | 설명 |
|---|---|
| **김연세** | 플레이어의 한국인 친구. 홍대 카페에서 처음 만남. |
| **플레이어 (나)** | 한국어를 배우는 외국인. 이름은 게임 시작 시 직접 입력. |

### 이름 처리 원칙

- 게임 시작 전 **이름 입력 화면**에서 한국어/영어 이름 입력
- 입력된 이름은 진입 대화 및 홍대 첫 퍼즐의 `[이름]` 토큰을 치환
- **받침 자동 감지**: 이름의 마지막 음절 받침 유무로 `이에요/예요` 자동 선택
  - 받침 있음 (민준) → `민준이에요.`
  - 받침 없음 (유리) → `유리예요.`
- localStorage에 이름 저장 → 재방문 시 이름 입력 화면 스킵

### 스토리 설정

서울을 처음 방문한 외국인 학습자가 홍대 카페에서 김연세를 만나 친구가 된다. 이후 서울 곳곳을 함께 여행하면서 실제 상황에서 한국어로 대화하는 경험을 쌓는다.

> **중요:** 홍대(1번 장소)는 **처음 만나는 장면**이므로 존댓말 사용.  
> 이후 장소에서 김연세는 친해진 사이로 반말과 존댓말을 혼용한다.

### 여행 경로

```
홍대 카페 → 명동 → 경복궁 → 한강공원 → 광장시장
→ 성수동 → 국립중앙박물관 → 북촌한옥마을 → DDP → 북한산
```

## 4. 화면 구성 및 플로우

### 전체 화면 흐름

```
앱 시작
  └→ [이름 입력 화면]  ← localStorage에 이름 있으면 스킵
        └→ (이름 입력 후 "시작하기")
              └→ [지도 화면]
                    └→ (장소 탭)
                          └→ [진입 카드 화면]
                                └→ (시작하기)
                                      └→ [퍼즐 화면] ← (힌트 / 카드 탭)
                                            ├→ (정답) → XP 토스트 + 스트릭 → [다음 문장]
                                            └→ (오답) → 피드백 버블 → [다시 풀기 / 다음으로]
                                                  └→ (마지막 문제 완료)
                                                        └→ 🎊 컨페티 + 팡파레
                                                              └→ [장소 완료 화면]
                                                                    └→ (지도로 돌아가기)
                                                                          └→ [지도 화면] (다음 장소 해제)
```

### 화면별 상세

#### 4-0. 이름 입력 화면 *(신규)*

- 전체 네이비 배경 (몰입감 있는 시작 경험)
- 🗺️ 이모지 + "서울 여행에 오신 것을 환영해요!" 타이틀
- 이름 입력 필드 (한국어/영어 모두 허용, 최대 20자)
- Enter 키 또는 버튼으로 제출
- 이름 없으면 버튼 비활성화
- 제출 시 이름 localStorage 저장 → 다음 방문부터 지도 화면 직접 진입

#### 4-1. 지도 화면

- 헤더: "**[이름] 씨의 서울 여행 🗺️**" + 누적 XP 배지 + 진행 현황 (`N / 10 장소 완료`)
- 서울 일러스트 SVG 지도 (viewBox: `50 22 280 170`)
- 핀 상태: **완료** (초록 #0f9b82) / **진행 가능** (빨강 #e03e3e + 박동 애니메이션) / **잠금** (회색, opacity 0.45)
- 완료 장소 간 경로선 (teal 점선)
- 지도 아래 장소 목록 카드 (스크롤)
  - 각 카드: 장소번호 뱃지 + 장소명 + 설명 + **연세 1권 N과** 태그(인디고) + 문법 배지(amber)
  - 완료 장소: **"복습하기 →"** 표시, 재진입 가능
  - 잠금 장소: 비활성화 (직전 장소 완료 시 해제)

#### 4-2. 진입 카드 화면

- 미니 지도 (viewBox: `65 5 270 112`, 160px) + "← 지도" 뒤로가기 버튼
- 장소 번호 + 이름 + 설명
- 김연세와의 **진입 대화** (장소별 고유, `[이름]` 토큰 치환)
- "이번 장소에서 배울 문법" 배지 카드
- "시작하기 →" 버튼

#### 4-3. 퍼즐 화면

- 미니 지도 (viewBox: `65 5 270 98`, 140px) + "← 장소 정보" 뒤로가기 버튼
- **스테이지 바**: "N번째 장소 — 장소명" + 스트릭 배지 (2연속↑ 시 🔥N연속) + 누적 XP pill
- 진행 바: 문제 수만큼 세그먼트 (완료: teal / 현재: navy / 미완: gray)
- **채팅 말풍선 영역** (스크롤, 정답/오답 후 자동 스크롤)
  - 친구 말풍선: 회색 배경, 좌측 "친" 아바타
  - 내 말풍선: navy 배경, 우측 정렬
  - 🌐 버튼: 인라인 배치, 탭하면 말풍선 안에 번역 표시
  - 정답/오답 후 피드백 버블 자동 추가 (정답: teal, 오답: 빨강 + 정답 표시)
- **퍼즐 카드**: 상황 힌트 텍스트 + 슬롯 영역
  - 문법 배지: 탭하면 **문법 팁 패널 토글** (▸닫힘 / ▾열림)
  - 슬롯 내 카드 탭: 트레이로 복귀
  - 정답: teal 칩 + bounce 애니메이션 / 오답: 빨강 배경 + shake 애니메이션
- **트레이**: 어절 카드 (무작위 셔플, 사용된 칩은 반투명 비활성)
- **버튼 바**: `힌트 (N)` + `확인하기` → 정답 후 `다음 문장 →` / 오답 후 `다시 풀기` + `다음으로 →`

#### 4-4. 장소 완료 화면

- 🎊 컨페티 파티클 폭발 (40개, 2초간 유지)
- 완료 히어로: ✓ 뱃지 + 장소명 + "완료!" + 이번 장소 획득 XP
- 통계 카드 3개: `정답 N/N` / `힌트 사용 N회` / `누적 XP`
- "이 장소에서 배운 문법" 복습 칩
- "지도로 돌아가기 →" 버튼 (teal)
- "다시 풀기" 버튼 (ghost)

### 장소 1 — 홍대 카페 (4–5과)

**진입 대화**
> 김연세: 안녕하세요! 저는 김연세예요. 😊  
> 나: 안녕하세요! 저는 [이름]이에요/예요. *(받침 자동 선택)*  
> 김연세: 반가워요! 같이 카페 가요!

**핵심 문법:** 이에요/예요 · 은/는 · 이/가 아니에요

| # | 상황 | 대화 | 정답 문장 | 오답 선택지 |
|---|---|---|---|---|
| 1 | 처음 만나 이름을 소개하는 상황 | 김연세: 안녕하세요! 저는 김연세예요. | 저는 [이름]이에요. *(받침 자동)* | 저가 / [이름]예요? / 이름이에요. |
| 2 | 서로 직업을 소개하는 상황 | 김연세: 직업이 뭐예요? | 저는 학생이에요. | 저가 / 학생이에요? / 의사예요. |
| 3 | 직업이 의사라고 소개하는 상황 | 김연세: 저는 의사예요. 직업이 뭐예요? | 저는 의사예요. | 저가 / 의사이에요? / 학생이에요. |
| 4 | 미국 사람이 아니라고 부정하는 상황 | 김연세: 미국 사람이에요? | 아니요, 미국 사람이 아니에요. | 네, / 사람이에요. / 아니에요? |

**문법 팁**
- `N이에요/예요`: 자음 끝 → 이에요 / 모음 끝 → 예요
- `은/는`: 자기 소개의 주제 표시 (저**는**)
- `이/가 아니에요`: 아니요 + N이/가 아니에요

### 장소 2 — 명동 (5–6과)

**진입 대화**
> 김연세: 명동에 왔어! 여기 쇼핑 천국이야 🛍️  
> 나: 와, 가게가 정말 많아요!  
> 김연세: 뭐 사고 싶어? 같이 골라보자.

**핵심 문법:** 이/가 아니에요 · 이/그/저

| # | 상황 | 대화 | 정답 문장 | 오답 선택지 |
|---|---|---|---|---|
| 1 | 멀리 있는 가방을 가리키는 상황 | 김연세: 저기 가방 가게 있어! | 저것은 제 가방이에요. | 이것은 / 가방이에요? / 저것이 |
| 2 | 가방이 내 것이 아니라고 말하는 상황 | 김연세: 이 가방 어때? 내 거야. | 이것은 제 가방이 아니에요. | 이것이 / 가방이에요. / 아니에요? |
| 3 | 저 사람이 의사가 아니라고 말하는 상황 | 김연세: 저 사람이 의사야? | 존슨 씨는 의사가 아니에요. | 존슨 / 씨는 / 의사가 / 아니에요? |
| 4 | 물건이 교과서인지 확인하는 상황 *(수정됨)* | 김연세: 이것 교과서예요? | 이것은 교과서가 아니에요. | 이것이 / 교과서예요. / 아니에요? |

> **[수정 이력]** 4번: 기존 "이 교과서 샤오밍 씨 거예요?" 질문은 답과 불일치 → "이것 교과서예요?"로 변경 (질문이 종류를 묻고, 답도 종류를 부정)

**문법 팁**
- `이것/그것/저것`: 이것 (here) / 그것 (near you) / 저것 (over there)
- `N이/가 아니에요`: 자음 끝 → 이 아니에요 / 모음 끝 → 가 아니에요

### 장소 3 — 경복궁 (7과)

**진입 대화**
> 김연세: 경복궁 도착! 여기서 한복도 빌려 입을 수 있어 😊  
> 나: 정말요? 완전 좋아요!  
> 김연세: 근데 먼저 주변 좀 파악해보자.

**핵심 문법:** ~에 있어요 · 도

| # | 상황 | 대화 | 정답 문장 | 오답 선택지 |
|---|---|---|---|---|
| 1 | 화장실 위치를 묻는 상황 | 김연세: 화장실 찾아야 해! | 화장실이 어디에 있어요? | 화장실을 / 어디가 / 없어요? |
| 2 | 연필과 볼펜이 모두 있는 상황 | 나: 볼펜도 있어요? | 연필이 있어요. 볼펜도 있어요. | 연필을 / 볼펜에 / 없어요. |
| 3 | 식당 위치를 알려주는 상황 | 김연세: 배고프지 않아? 식당 찾아보자. | 식당이 서점 근처에 있어요. | 식당을 / 근처가 / 있어요? |
| 4 | 가방 속 지갑을 확인하는 상황 | 김연세: 가방에 지갑 있어요? | 지갑이 가방에 있어요. | 지갑을 / 가방을 / 있어요? |

**문법 팁**
- `N이/가 어디에 있어요?`: 장소를 물을 때
- `N도`: 앞 내용에 추가할 때 (볼펜**도** 있어요)
- `N이/가 장소에 있어요`: 위치를 말할 때

### 장소 4 — 한강공원 (8과)

**진입 대화**
> 김연세: 한강이다! 여기서 치킨 먹어야지 🍗  
> 나: 저도 배고파요!  
> 김연세: 자리 잡고 주문해보자.

**핵심 문법:** 아요/어요/여요 · 을/를

| # | 상황 | 대화 | 정답 문장 | 오답 선택지 |
|---|---|---|---|---|
| 1 | 한강공원에서 김밥을 먹는 상황 | 김연세: 한강이다! 치킨 먹자! | 점심에 김밥을 먹어요. | 저녁에 / 김밥이 / 먹어요? |
| 2 | 한국어 공부에 대해 말하는 상황 | 김연세: 한국어 공부해요? | 저는 한국어를 공부해요. | 저가 / 한국어가 / 공부해요? |
| 3 | 친구가 책 읽는 모습을 보는 상황 | 김연세: 저기 봐봐! 친구가 책 읽고 있어. | 친구가 책을 읽어요. | 친구는 / 책이 / 읽어요? |
| 4 | 카페에서 커피를 마시는 상황 | 김연세: 저기 카페 보여요? 커피 마셔요? | 저는 커피를 마셔요. | 저가 / 커피가 / 마셔요? |

**문법 팁**
- `을/를`: 자음 끝 → 을 / 모음 끝 → 를 (김밥**을**, 커피**를**)
- `공부하다 → 공부해요` (하다 → 해요)
- `마시다 → 마셔요` (ㅣ 모음 축약)

**핵심 문법:** -지 않다 · ㅂ불규칙

| # | 상황 | 대화 | 정답 문장 | 오답 선택지 |
|---|---|---|---|---|
| 1 | 김치를 안 먹는다고 말하는 상황 | 김연세: 여기 김치 한번 먹어봐! | 저는 김치를 먹지 않아요. | 저는 / 김치가 / 먹지 / 않아요? |
| 2 | 음식이 맵지 않다고 말하는 상황 | 김연세: 여기 음식 매워 보여. 괜찮아? | 한국 음식이 맵지 않아요. | 한국 / 음식은 / 맵지 / 않아요? |

**문법 팁 (ㅂ불규칙 주의사항)**
- `동사/형용사 어간 + -지 않아요`: 부정 표현
- **ㅂ불규칙 적용 범위**: **모음 어미 앞에서만** 발동
  - 모음 앞: 맵다 → **매**워요 (ㅂ→w) / 덥다 → **더**워요
  - 자음 앞(`-지`): 맵**지** 않아요 / 덥**지** 않아요 ← ㅂ 그대로 유지

**핵심 문법:** -을까요? · 하고

| # | 상황 | 대화 | 정답 문장 | 오답 선택지 |
|---|---|---|---|---|
| 1 | 음료를 제안하는 상황 | 김연세: 여기 유명한 카페야! 뭐 마실까? | 아이스 아메리카노 마실까요? | 아이스 / 아메리카노가 / 마실까요. |

**문법 팁**
- `동사 어간 + -을까요?/-ㄹ까요?`: 제안 (먹다 → 먹**을까요?**)
- `N하고 N`: 구어체 "and" (커피**하고** 물)

## 6. 퍼즐 메커닉

### 어절 카드 조립 방식

1. 정답 어절 카드 + 오답 어절 카드를 **무작위 셔플**하여 트레이에 표시
2. 학습자가 **순서대로** 카드를 탭 → 슬롯에 쌓임
3. 슬롯 내 카드 탭 → **제거** (트레이로 복귀)
4. "확인하기" 탭 → 정답/오답 판정 + 사운드 + 햅틱

### 오답 선택지 설계 원칙

- 정답 카드와 어떻게 조합해도 **올바른 한국어 문장이 되지 않는 것**만 허용
- 틀린 조사, 다른 격조사, 의문형 어미(-에요?) 등으로 변별
- **불가능한 조합 금지**: 모음 어미 명사 뒤 `이/을/과` 불가, 자음 어미 명사 뒤 `가/를/와` 불가

### 힌트 시스템

- 장소당 힌트 3회 제공 (`hintsLeft` 상태로 관리)
- 힌트 사용 시: **다음 정답 어절 자동 배치** (슬롯에 직접 추가)
- 힌트 사용 여부에 따라 XP 차등 지급 (힌트 사용: +10 XP / 미사용: +20 XP)

### 피드백 시스템

| 상태 | 슬롯 border | 칩 스타일 | 애니메이션 |
|---|---|---|---|
| 기본 | 회색 점선 | navy 배경, 흰 글씨 | — |
| 정답 | teal 실선 | teal 배경 (`.cor`) | `sp-chipBounce` (60ms 스태거) |
| 오답 (해당 위치) | 빨강 실선 | 빨강 배경 + 취소선 (`.wrg`) | `sp-chipShake` |
| 오답 (올바른 위치) | — | teal 배경 (`.cor`) | — |

## 7. XP 및 스트릭 시스템

### XP 지급

| 조건 | XP |
|---|---|
| 정답 (힌트 없음) | +20 XP |
| 정답 (힌트 사용) | +10 XP |
| 오답 후 다음으로 | +5 XP |

- XP는 localStorage에 영구 누적
- 토스트 형식: `⭐ +20 XP!` — 스트릭 시 `⭐ +20 XP! 🔥3연속`

### 스트릭 시스템 *(신규)*

- 연속 정답 시 `streak` 카운터 증가
- 오답 시 즉시 리셋 (0으로)
- 2연속 이상 시 스테이지 바에 **🔥N연속** 배지 표시 (pulse 애니메이션)
- "다시 풀기"는 스트릭에 영향 없음 (새 시도로 카운트)

## 8. 사운드 & 피드백

### Web Audio API 사운드 (외부 파일 없음)

모든 사운드는 Web Audio API로 실시간 합성. 첫 사용자 제스처에서 AudioContext 초기화 (자동재생 정책 준수).

| 이름 | 트리거 | 합성 방식 |
|---|---|---|
| `playTap()` | 트레이 칩 탭 | sine 600Hz, 0.06s |
| `playCorrect()` | 정답 판정 | sine C5(523Hz)→E5(659Hz) 두 음 |
| `playWrong()` | 오답 판정 | sawtooth 260→180Hz, 0.18s |
| `playXp()` | XP 토스트 | sine 아르페지오 G4→C5→E5→G5 (4음) |
| `playComplete()` | 장소 완료 | sine 팡파레 C5→E5→G5→C6 (4음) |

### 햅틱 피드백 (`navigator.vibrate`)

| 상황 | 패턴 |
|---|---|
| 칩 탭 | `10ms` |
| 정답 | `[15, 50, 15]` |
| 오답 | `[30, 20, 30]` |
| 장소 완료 | `[50, 100, 50, 100, 100]` |

### TypeScript 타입

```typescript
type Screen   = 'name' | 'map' | 'entry' | 'puzzle' | 'complete';
type NavDir   = 'forward' | 'back';

interface Location {
  id: string; name: string; num: number; x: number; y: number;
  unit: string; desc: string; grammar: string[];
  entryMessages: { type: 'friend' | 'self'; text: string }[];
}

interface Puzzle {
  friendMsg: string;  friendMsgT: string;
  selfMsg: string | null;  selfMsgT: string | null;
  friendMsg2: string | null;  friendMsg2T: string | null;
  hintText: string;
  answer: string[];       // 정답 어절 배열 (순서 중요)
  distractors: string[];  // 오답 어절 배열
  grammar: string;
  tip: string;            // HTML 허용 (<strong>, <br>)
}

interface SavedState {
  playerName: string; completed: string[];
  totalXp: number; currentLoc: string | null;
}
```
